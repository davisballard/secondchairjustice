## Critical Issues (Fix These First)

### Issue #1: Mobile Hero Section Kills Conversions
**Problem:** Hero headline is 102.4px on mobile ‚Äî takes up entire screen. CTA button is below fold. Users must scroll to take action.

**Impact:** Losing 60%+ of mobile conversions

**Fix:**
- Reduce H1: 102.4px ‚Üí 56-64px on mobile
- Move CTA above fold OR add sticky bottom CTA bar
- Increase CTA button height: 48px ‚Üí 56px

**Expected Lift:** +20-35% mobile conversion rate

---

### Issue #2: Scrolling Proof Bar Creates Noise
**Problem:** 7 stats scrolling horizontally = cognitive overload + visual distraction

**Fix:**
- Replace with static 2√ó2 grid
- Show 4 key stats: 92%, $1,600, 47 territories, 0 shared

**Expected Lift:** +10-15% comprehension and trust

---

### Issue #3: No Visual Proof of Exclusivity
**Problem:** "Geo-exclusive" is told, not shown. Abstract concept needs visualization.

**Fix:**
- Territory map showing US with green dots (locked) and gray outlines (available)
- Interactive or static
- Place after "One firm per market" section

**Expected Lift:** +20-25% engagement with geo-exclusive concept

---

### Issue #4: CTA Below Fold on Mobile
**Problem:** Primary conversion action requires scrolling. Friction kills conversions.

**Fix:**
- Tighten hero section to bring CTA above fold
- OR: Add persistent sticky CTA bar at bottom of screen

**Expected Lift:** +25-35% CTA visibility and clicks

---

### Issue #5: Too Many Competing CTAs
**Problem:** 20 clickable elements, 1 goal = 20:1 attention ratio (should be 3:1 max)

**Fix:**
- Remove "Explore vertical" links from homepage
- De-emphasize footer navigation
- Keep 2 primary CTAs only

**Expected Lift:** +15-20% focused conversions

## How to Implement: Recommended Workflow

### Lovable = Primary Tool (95% of Work)
Use Lovable for:
- Mobile responsive CSS changes
- Component-level changes (proof bar, CTA buttons)
- New components (sticky CTA bar, territory map container)
- Layout adjustments
- Interactive features (quiz, map interactivity)
- **ALL COPY CHANGES** (headlines, body text, CTAs, new sections)

### Framer = Final Polish Only (5% of Work)
Use Framer ONLY for:
- Micro-interactions after everything works
- Fine-tuning hover states
- Animation easing curves
- Typography kerning (if needed)

**Best Practice:** Implement copy changes in Lovable. Framer is finishing touches only.

### Recommended Team Structure

**Implementation Lead:**
- Prompt Lovable for structure AND copy changes (see `08_Lovable_Prompts.md`)
- Test outputs on mobile after each change
- Drive Phase 1-2 mobile optimization and content
